Release notes of GoCD

This page contains the release notes for the next version of GoCD

  • (latest)

(next)

UX Changes

  • 67f1605 - Find Value Stream Map (VSM) more easily. (#3705) (#3858)

    9f6909e - Renames pipeline 'Label' to 'Instance' as part of an ongoing UX refresh. (#3944) (#3949)

    Pipeline instances have an explict link to VSM across the application for better discoverablity. Moreover, the term pipeline Label has been changed to Instance as part of an ongoing UX refresh.

    Page headers refreshed across application

Other Improvements

  • eb7305e - Improves classloading and resource management. (#3792)

    Prevents file-descriptor and classloader leaks by ensuring that:

    • Classloaders for agent-launcher and agent are closed properly
    • Temporary jar files that are extracted are removed after closing the classloaders
    • File watcher from log4j is replaced with a custom implementation
  • f501efc - Move to logback for logging. (#3831)
    • As a part of this change, *-log4j.properties have been moved over to *-logback.xml.
    • This migration solves a number of logging related issues such as #3588, #3557 and #3403
  • a8d09f3 - Provides support for parameters and template references in config repo pipelines.
  • 36b675e - Introduces support for bitbucket webhook.
  • 6320d55 - Introduces support for gitlab webhook.
  • 2cb4c00 - Makes Go Server session timeout configurable. (#3856)
  • 4e5eeda - Updates footer links to point to installed version's api and docs pages. (#3837)
  • e51f3b6 - GoCD Agent supports configuring proxies. (#995)
  • c7ec496 - Links pipeline group names in dashboard to corresponding pipeline group settings page for an admin or pipeline group admin. (#3795)

Bug Fixes

  • cef6649 - Fixes a race condition during Job reschedule. (#3891)
  • a1f9945 - Improves checksums for agent jar(s) download. (#3884)
  • 4db7e52 - Does not show the settings icon for config repo pipelines on VSM. (#3892) (#3881)
  • 686c76d - Fixes issue with users authenticated using WebBasedAuthentication not assigned the right permissions. (#3878)

API Changes

  • cba778c - Introduces health check API for Server. (#3912)
  • 97df2bf - Introduces health check api for agent. (#3781) (#3074)
  • e4d041b - Adds v2 of Users API. (#3720)

Docker Images

  • We have made a few changes to the GoCD Server and the GoCD Agent Docker images.

Breaking Changes

  • Support for phusion based docker server and agent images have been removed this release. We provide support for the docker images for GoCD agents across the following operating systems - Alpine 3.5, CentOS 6 & 7 , Debian 7 & 8, Ubuntu 12.04, 14.04 & 16.04

Deprecations

  • As of release 17.9, version 2 of the plugin info API is deprecated. This version of the API will be removed in 17.12.0 release (scheduled to be released in November 2017). Version 3 of the API is available, and users are encouraged to use it.
  • As of release 17.5, authentication plugin endpoints have been deprecated. Support for these plugins will be removed in 18.1 (scheduled to be released in January 2018). Plugin developers are encouraged to migrate their plugins to use the Authorization Plugins that were introduced. The following plugins are affected by this deprecation:

    • Google OAuth Login (Alternative: Google OAuth Authorization Plugin)

    • GitHub OAuth Login (Alternative: GitHub OAuth Authorization Plugin)

    • GitLab(CE) OAuth Login (Alternative: GitLab OAuth Authorization Plugin)

    • TLS Client Certificates Login (Use latest version of TLS Client Certificates Login instead.)

    • Strong Auth Plugin

    • Guest Login

  • 8571bc1 - Removes garage controller and associated services. (#3923)

Note

  • With release 17.5.0 support for in-built password file and LDAP/AD authentication had been deprecated. In this release support for this has been removed in favour of the bundled LDAP and Password File plugins respectively.
